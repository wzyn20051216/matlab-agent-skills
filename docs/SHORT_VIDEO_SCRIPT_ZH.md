## 四、90~120 秒增强版

大家好，我最近在做一个比较硬核的项目，叫 `matlab-agent-skills`。  
它本质上是一套面向 MATLAB / Simulink 的 AI 工程技能仓库。

很多人理解 AI + MATLAB，还停留在“帮我写个函数、补个脚本、讲讲原理”这个阶段。  
但真实工程不是这样的。  
真实工程要的是闭环：你要能运行、能仿真、能导出结果、能代码生成、能查日志、能验收，最好还能接上 STM32、Raspberry Pi、CubeMX、Keil、VS Code 这些开发链。

所以我做这个仓库，核心不是“让 AI 更会说 MATLAB”，而是“让 AI 真能把 MATLAB 工程跑起来”。

为什么我觉得这个方向特别适合嵌入式、电气、自动化、电力电子和程序员？  
因为这些行业的人，日常工作本来就跨很多层：

- 上层要做算法验证和数据分析；
- 中间要做 Simulink 建模和系统仿真；
- 下层要做代码生成、嵌入式部署和工具链联调；
- 最后还要做交付、复现和验收。

而 MATLAB R2026a 这一版，官方在几个方向上特别有价值。  
第一，建模和协作更智能了，比如 Simulink Copilot。  
第二，硬件和部署链更实用了，比如 Raspberry Pi Blockset 和 STM32 Microcontroller Blockset。  
尤其是从 R2026a 开始，这两条线在官方体系里都更明确地进入 blockset 形态，更适合往工程闭环和产品化流程上走。  
第三，代码生成与跨语言协作链路更适合工程自动化。  
第四，MATLAB 启动性能和 Python 协同这些基础体验也在继续增强。

我这个 Skills 仓库，就是把这些能力统一封装。  
比如你丢给它一个任务：帮我验证一个控制算法，从 MATLAB 数据处理开始，到 Simulink 仿真，再到 C 代码生成，再检查 STM32 开发链能不能接上。  
它就会按流程去跑，而不是只给你一个“理论上可以”。

而且这套仓库最重要的一点是：我特别强调闭环验收。  
每个任务尽量都要有命令入口、日志、图表、代码产物、JSON 报告，最后告诉你这个任务到底通过没通过。

所以如果你是工程师，这套东西的意义非常直接：  
它不是帮你少敲几行代码，而是在把你原来很碎、很依赖手工的 MATLAB 工程流程，逐步自动化、标准化、可复现化。

## 五、Demo 演示口播稿

> 这一段适合你边录屏边讲，配合终端、README、MATLAB、生成产物目录一起展示

我给大家演示一个最典型的场景。  
假设我是一个做嵌入式控制或者自动化系统的人，我现在要验证一条 MATLAB 到嵌入式工具链的流程。

以前我可能要自己一项一项确认：  
MATLAB 能不能启动？  
Simulink 能不能跑？  
代码生成能不能过？  
STM32 支持包装没装？  
Raspberry Pi 支持包有没有？  
CubeMX、Keil、VS Code 能不能联起来？

现在我直接跑一条验收命令就行。

```powershell
$matlabExe = (Get-Command matlab -ErrorAction SilentlyContinue).Source
powershell -ExecutionPolicy Bypass -File .\scripts\Test-MatlabEmbeddedStack.ps1 -MatlabPath $matlabExe
```

这条命令会去做几件事：

- 拉起 MATLAB；
- 检查工具箱和关键能力；
- 跑 MATLAB / Simulink / Coder 的验证；
- 探测 STM32 和 Raspberry Pi 硬件支持；
- 检查 CubeMX、Keil、VS Code 和关键扩展；
- 最后把结果输出成结构化报告。

最后你会拿到的，不是一句“应该没问题”，而是一套真实产物。  
比如日志、JSON 验收报告、功能探针结果、编译链结果。

这对工程师意味着什么？  
意味着你以后做项目，不管是控制算法、自动化系统、嵌入式部署，还是论文复现，都能先用一套标准流程把环境和能力校验干净。  
这就相当于把“拍脑袋开发”，变成“带证据开发”。
